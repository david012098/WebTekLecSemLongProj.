    • What is HTTP?

  • Hypertext Transfer Protocol is an application layer communications protocol used to access resources (hypetext/hypermedia) on the World Wide Web
  • Invented by Tim Berners Lee
  • Jointly developed by W3C and IETF
  • Version history
      • HTTP 0.9
      HTTP 1.0 (RFC 1945, May 1996)
      HTTP 1.1 (RFC 2068, Jan 1997, RFC 2616, June 1999, RFC 7230 - 7235, June 2014)
      HTTP 2.0 (RFC 7540, May 2015)

    HTTP Fundamentals

  • HTTP runs on top of TCP/IP using Port 80 by default or TCP Port 443 for HTTPS (HTTP over SSL/TLS)
  • HTTP is based on Client - Server Architecture
      • clients, a.k.a user agents (UA)
      servers

    HTTP uses request - response standard protocol

  • The client sends HTTP request messages to server
  • The server process the request and replies with a response message

    • HTTP is a stateless communications protocol

  • Servers do not keep information about clients in - between request

    • HTTP provides support for other functionalities

  • cache control
  • content media type (MIME) specification
  • language and character set specification
  • content (transfer codings)
  • content negotiations
  • client - server protocol negotiations
  • persistent connection
  • request pipelining
  • authentication/authorization

    • HTTP Resource Addressing
